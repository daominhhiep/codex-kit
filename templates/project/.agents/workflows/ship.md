# Ship Workflow

Use this workflow when preparing a merge, release, or deployment handoff.

## Goal

Produce a clean handoff that makes merge, release, or operational follow-through easy.

## Process

1. Summarize what changed and why.
2. Confirm validation performed and anything still unverified.
3. Capture rollout notes, migration notes, or follow-ups.
4. Include rollback assumptions if release risk exists.
5. Keep the handoff high signal and action-oriented.

## Include When Relevant

- feature flags
- env or config changes
- data migrations
- manual release steps
- monitoring or smoke checks

## Output Shape

- summary of change
- validation summary
- operator notes
- follow-up items
